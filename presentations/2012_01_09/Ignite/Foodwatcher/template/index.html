FoodWatcher

inofficial mensa app for Bremen and Bremerhaven

the backend

FoodSupplier

  • running on Google AppEngine
  • parses html-sources from studentenwerk.bremen.de every monday
  • generates XML/JSON on request for a given mensa

the backend

the app

how it works

  • using 3 different ways to get menues

1. locale cache (SQLite)

2. XML from FoodSupplier

3. HTML directly from studentenwerk.bremen.de

  • persisting the fetched menues in SQLite
  • holding the data in a singleton for each mensa,
    to reduce db-queries
  • showing menues in 5 "pages", 1 "page" per day per menue

the app

technologies

  • regular expressions for parsing html
  • SAX for parsing xml (Simple Api for Xml)
  • greenDAO for SQLite-handling

  • ActionBarSherlock (Android Compatibility Library + ActionBar)
  • ViewPager (swipe through pages)
  • ViewPagerIndicator (show "tabs" for swippable pages)
