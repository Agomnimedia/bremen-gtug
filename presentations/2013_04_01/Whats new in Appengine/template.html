New stuff in GAE

Expanded EU support

  • 13th December 2012 - Release of GAE 1.7.4
  • Expanded EU support
    • Deployment of your GAE application to the European Union Datacenters
    • Ireland, Finland and Belgium
    • Advantage: Smaller request times
  • Only available for user which have enabled billing for their applications
  • Further support will be coming in the next months
  • Link to the 1.7.4 Release Notes

Console expansions

Memcache Viewer

  • Shows cache size
  • Shows hit and miss count
  • Shows item count
  • Shows age of oldest item
  • Action to flush the cache
  • Action to search for an memcache item

Console expansions

Datastore Statistics

  • Shows total size used by entity
  • Shows average size use by one entity entry
  • Shows amount of entries per entity
  • Shows property type information of entity
  • Shows attribute informations of entity

New APIs

API lifecycle

  • API only for Trusted Testers (optional)
    • API can only be used by a small group of developers
  • API is experimental
    • API can have bugs
    • Chance of not getting the API to stable
  • API is stable
  • API is deprecated
    • New Deprecation Policy introduced in April 2012
    • The API will exist for one more year and then be deleted

New APIs

App Identity

  • Experimental
  • Available for Java and Python
  • Guarantees the identity of the GAE application
  • Example: I only want my application 'test1' to be able to send a request to my other application 'test2'.

New APIs

Capabilities API

  • Stable
  • Available for Java, Python and Go
  • 'Your application can detect outages and scheduled downtime for specific API capabilities'
    • CapabilitiesService service =
    CapabilitiesServiceFactory.getCapabilitiesService();
CapabilityStatus status = 
    service.getStatus(Capability.IMAGES).getStatus();

if (status == CapabilityStatus.DISABLED) {
    // Images API is not available.
}

New APIs

Google Cloud Endpoints

  • Experimental
  • Available for Java and Python
  • 'Google Cloud Endpoints consists of tools, libraries and capabilities that allow you to generate Endpoints and client libraries from an App Engine backend to simplify client access to that web app'
  • A talk will follow ;-)

New APIs

Prospective Search

  • Experimental
  • Available for Java and Python
  • 'Allows an application to register a large set of queries that are simultaneously matched against a stream of input documents'

New APIs

Google Cloud Storage

  • Experimental
  • Available for Java
  • Cloud Storage is used to store and serve large files
  • Cloud Storage has no file size limit
  • Can only be used if Billing is enabled

Google Cloud Messaging

  • Formerly known as C2DM - Cloud to Device Messaging
  • Pushes messages from your application server to android devices
  • Can be used on any application server

Find more details by seeing the video of the GDG Bremen regulars' table last december

Java 7 and Maven

  • Java 7 is supported since GAE 1.7.5
  • 'In a future release, support for Java 6 will be removed'
  • New Apache Maven Plugin for GAE (since GAE 1.7.4)
    • <plugin>
    <groupId>com.google.appengine</groupId>
    <artifactId>appengine-maven-plugin</artifactId>
</plugin>

Check out the 'Maven' talk by +Cornelius Lilge at the Hackerspace Bremen on 30th april.
